Initializing Pseudoforums

Hello, this is the creator of Pseudoforums speaking and I will now take you through a tour to initialize this platform you are now using called Pseudoforums.

First of all, feel free to scroll down the HTML code of this page (and every other page provided by this platform) and edit the copyright notice at the footer. This project is MIT Licensed, which means that as long as you keep the original copyright notice somewhere, you can do just about anything with it.

After that, you should edit the database located at pf-assets/js/global.js to include your own account. Preferably, if you are the one hosting a GitHub Forum using this platform, you should be the Administrator of this Forum and thus your status should be 7. The pf_database located at pf-assets/js/global.js is an array of objects (accounts). For example: 

var pf_database = [
	{
		username: "pfadmin",
		password: "pf9487",
		status: 7,
		name: "Pseudoforumsadministrator"
	},
	{
		username: "pfdev",
		password: "devedev",
		status: 6,
		name: "Pseudoforumsdeveloper",
		profilePic: "pf-images/dev.jpg"
	}
];

The above example shows 2 accounts. The first account has username "pfadmin" and password "pf9487", which is (obviously) the details required for the user to login to that account. The first account has a status of 7, which means that he/she is an Administrator of the Forum and his/her displayed name on Forums when posting is "Pseudoforumsadministrator". The second account has login details username: "pfdev" and password: "devedev", a status of 6 (Developer), a displayed name of Pseudoforumsdeveloper and a custom profile picture with path "pf-images/dev.jpg".

At this point, you may have noticed that the first account does not have the profilePic property while the second account does. In this Forum platform, accounts must have the following details (properties): username, password, status and name. The profilePic is optional and for those who have custom profile pictures. If the profilePic property is not specified in the account details, the code will automatically generate a default profilePic with the path pf-images/system/default.jpg.

Initially, this platform comes with 8 dummy accounts (view the database to view the account details). When you start using this platform, you should delete the dummy accounts and add your own account(s) instead.

After you have configured your database, you should then configure the email address that the commenting, posting and moderating requests will be sent to. Unfortunately, each panel operates on their own Javascript file so you will have to modify the email thrice. If you go to comment.js, post.js and mod.js, you should see a variable called pf_adminEmail (or equivalent) declared at the very start of the file. You should change the value of the variable to your own email address. The rest of the code will sort it out. You can also customize the subject of the email you will receive (follow comments in those JS files).

Well, I think this is it. If you have any further questions or would like to report bugs/glitches on this platform, feel free to email dleung@connect.kellettschool.com to report it. Enjoy!

Creating Threads, Closing Threads and Commenting

A quick introduction to creating, closing and commenting on threads to get you started. As you may have noticed by reading the code, all secure Forum content (i.e. content that can only be accessed by logging in) lies directly within the <span> tags with id="secure". In more technical terms, all secure Forum content are direct children of the tag mentioned above.

To create a normal thread:

  1. Create a div with class="row thread" directly within the <span id="secure"> tag like this: <div class="row thread"></div>

  2. Within the div you just created, insert the following code: <div class="4u 12u$(medium)"></div><div class="8u 12u$(medium)"></div>

  3. Inside the div with class="4u 12u$(medium)", insert a div with class="<username_of_user_who_created_this_thread>". The code in this platform will sort the rest out for you.

  4. Finally, feel free to add content of any type within the div with class="8u 12u$(medium)". Ideally the content in this div should have a <h3> header and at least 1 paragraph.

To close a thread, simply change the class of the thread from row thread to row thread closed and add the following code at the start of the "h3" title: <span class="icon fa fa-lock" title="This thread is closed. You may no longer comment on this thread."></span>.

To comment on a thread, simply follow the instructions on creating a thread and then changing the class of the thread from row thread to row. Comments should ideally be directly under the parent thread.
